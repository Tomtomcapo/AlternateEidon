# Mort et Persistance

## Résumé

Comment fonctionne la mort dans la simulation — et pourquoi les Eidons sont différents.

## La mort ordinaire

Pour les habitants normaux de la simulation :
- La mort est **définitive**
- La conscience se dissipe
- Pas de résurrection possible
- Cycle normal de vie

## La mort des Eidons

Les Eidons sont **quasi-immortels** mais pas invulnérables.

### Ce qui ne peut pas les tuer
- Vieillissement (ils ne vieillissent pas)
- Maladie ordinaire
- Blessures mineures (régénération)
- Poisons normaux

### Ce qui peut les tuer
- Violence extrême et soutenue
- Destruction massive du corps
- Certaines armes ou méthodes spécifiques
- Un autre Eidon

### La mort d'Altert
[[characters/altert|Altert]] est mort (~980) :
- Circonstances exactes non clarifiées dans le canon
- Prouve que les Eidons **peuvent** mourir
- A déclenché le failsafe (création de Gamma)

## Le mécanisme de failsafe

### Principe
Si les Eidons disparaissent, la simulation risque l'instabilité. Le failsafe est une **mesure de sécurité** :
- Surveille l'état des Eidons
- Si conditions critiques → intervention
- Crée un nouvel Eidon de remplacement

### Comment ça fonctionne
Le failsafe ne copie pas une personne réelle (comme les Eidons originaux). Il **promeut** une conscience locale :
- Sélection d'un habitant existant
- Élévation au statut d'Eidon
- Transfert de capacités
- Pas de mémoire de l'opération

### Le cas Gamma
[[characters/gamma|Gamma]] est le produit du failsafe :
- Vivait dans l'[[locations/extreme-est|Extrême-Est]]
- Sélectionnée à la mort d'Altert
- Ne sait pas ce qu'elle est
- Découvre progressivement ses capacités

## Persistance post-mortem

### Cas normal
Rien ne persiste. La mort est la fin.

### Cas Axylum
[[characters/axylum|Axylum]] est un cas **anormal** :
- Entité massive créée par Seth
- Sa mort n'a pas libéré ses ressources
- Son essence persiste comme l'[[concepts/effluve|Effluve]]
- Bug ou feature ? Inconnu.

### Implications
Si Axylum a persisté partiellement, d'autres entités pourraient-elles aussi ?
- Les Eidons morts laissent-ils une trace ?
- Altert a-t-il disparu complètement ?
- Questions ouvertes.

## La réinitialisation

### Menace ultime
Si tous les Eidons meurent **et** le failsafe échoue, la simulation peut être **réinitialisée** :
- Retour à un état antérieur
- Perte de tout ce qui s'est passé depuis
- Nouvelle tentative

### Ce qu'Altert savait
Altert a découvert ce mécanisme dans [[locations/babel|Babel]]. C'est une des raisons pour lesquelles il a gardé le secret :
- Révéler la menace de réinitialisation = chaos
- Mieux vaut que personne ne sache

## Détails canoniques

1. Les mortels meurent définitivement
2. Les Eidons sont quasi-immortels mais peuvent mourir
3. Altert est mort (~980)
4. Le failsafe crée des Eidons de remplacement
5. Gamma est un produit du failsafe
6. Axylum persiste anormalement (Effluve)
7. La réinitialisation est possible mais catastrophique
