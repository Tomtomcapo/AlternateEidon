# Framework de spécification — Alternate Eidon

Ce dossier définit **comment** on structure, indexe et maintient le canon.

## Sources de vérité (canon)
- `universe/LORE.md` : genèse + entités centrales.
- `characters/` : fiches personnages (Eidons et autres).
- `plot/` : chronologie et intrigue (à construire).

## Comment ajouter du contenu
Utiliser les prompts prêts à l’emploi dans `.github/prompts/` :
- Ajout personnage
- Ajout lieu
- Ajout faction
- Ajout concept de lore
- Ajout élément de scénario / scène

Après ajout :
- mettre à jour `spec/MANIFEST.md`
- mettre à jour `universe/GLOSSARY.md` si un nouveau terme apparaît

## Principe clé
Un ajout n’est accepté en “canon” que s’il :
- ne contredit pas le canon existant,
- a des liens clairs avec le monde (Eidons / factions / lieux / époque),
- a un impact narratif identifié.
