# Conventions d’écriture et d’archivage

## Règles de nommage
- Personnages : `characters/<NomCanon>.md` (ex: `Maelith.md`).
- Lieux (si créés) : `universe/locations/<NomCanon>.md`.
- Factions (si créées) : `universe/factions/<NomCanon>.md`.
- Concepts de lore : `universe/concepts/<NomCanon>.md`.

## Statuts
- Par défaut, tout est **canon** si présent hors `drafts/`.
- Les brouillons vont dans `drafts/` (si besoin plus tard).

## Structure minimale par type
Voir les templates dans `spec/templates/`.

## Références croisées
- Préférer les phrases explicites plutôt que des listes de liens.
- Mentionner le document source quand un fait clé dépend d’un autre fichier.

## Cohérence
- Un fait clé (origine/mort/lien familial/lieu/autorité) doit être cohérent avec `universe/LORE.md`.
- Si une nuance est nécessaire : préciser “selon X” (point de vue de faction) plutôt que d’énoncer deux vérités incompatibles.
